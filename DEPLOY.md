# HackerLink Cloud Deploy

This app deploys as one Render web service:

- FastAPI serves `/api/*`
- FastAPI also serves the built React app from `frontend/dist`
- Render PostgreSQL stores app data
- Render Redis stores explore task status/results

## 1. Push This Repo To GitHub

Render Blueprint deploys from a GitHub repository.

## 2. Create The Render Blueprint

1. Open Render.
2. Choose **New +** -> **Blueprint**.
3. Connect the GitHub repo.
4. Select `render.yaml`.
5. Apply the blueprint.

Render will create:

- `hackerlink-app`
- `hackerlink-db`
- `hackerlink-redis`

## 3. Set Secrets

In the `hackerlink-app` service, set:

```text
GLM_API_KEY=<your real GLM/Zhipu API key>
```

`SECRET_KEY` is generated by Render. `DATABASE_URL` and `REDIS_URL` are wired from the database and Redis services.

## 4. Update CORS If You Rename The Service

If the Render service URL is not:

```text
https://hackerlink-app.onrender.com
```

update `CORS_ORIGINS` in Render to your actual URL. Same-origin browser usage still works, but keeping this accurate helps API calls from the deployed frontend.

## 5. Verify

Open:

```text
https://<your-render-url>/api/health
```

Then open:

```text
https://<your-render-url>/
```

On first startup, the backend initializes the PostgreSQL schema automatically from `backend/schema.sql`.
